Connect in 30 seconds

Point your MCP client at one URL

Add the server URL to your MCP client and sign in once in the browser — Google or Microsoft SSO. No key to mint, copy, or rotate. The agent connects over streamable HTTP; no install, no local process.

Sign in once — no API key to paste.

On the first connection your client opens a browser for a one-time SSO sign-in (Google or Microsoft) — standard OAuth. After that it reconnects and refreshes access on its own; you're not prompted again unless you disconnect.

Server URL
https://mcp.smplkit.com/api/mcp

Your client handles authentication for you: the first connection opens a browser to sign in, then it refreshes access in the background — no token to manage.

CLI
claude mcp add --transport http smplkit https://mcp.smplkit.com/api/mcp
…or in .mcp.json
{
  "mcpServers": {
    "smplkit": {
      "type": "http",
      "url": "https://mcp.smplkit.com/api/mcp"
    }
  }
}

On first use Claude Code opens your browser to sign in, then remembers the connection.

~/.cursor/mcp.json or project .cursor/mcp.json
{
  "mcpServers": {
    "smplkit": {
      "url": "https://mcp.smplkit.com/api/mcp"
    }
  }
}

Cursor prompts you to sign in the first time you use the server.

claude_desktop_config.json
{
  "mcpServers": {
    "smplkit": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://mcp.smplkit.com/api/mcp"]
    }
  }
}

Desktop bridges remote servers through mcp-remote, which opens a browser for the one-time sign-in and caches the connection.

Prefer a static key?

For non-interactive use — CI, scripts, or writing code with the smplkit SDKs — skip the browser and authenticate with an API key as a bearer token: add Authorization: Bearer YOUR_SMPLKIT_API_KEY to the server config (a custom X-Smplkit-Api-Key header also works). The SDKs read the same key from SMPLKIT_API_KEY.
